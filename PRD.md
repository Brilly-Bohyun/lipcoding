# PRD (Product Requirements Document) — Vendor Support RCA Copilot

## 1. 제품 비전

MSP 엔지니어가 벤더 서포트 메일 스레드에서 한국어 RCA 보고서를 15분 이내에 완성할 수 있게 하는 AI 기반 생산성 도구.

---

## 2. 대상 사용자

### Primary Persona: MSP 엔지니어 (최보현)

| 항목 | 내용 |
|------|------|
| 역할 | Azure/AWS 인프라 운영, 벤더 서포트 커뮤니케이션 |
| 일일 업무 | 벤더 서포트 티켓 3~5건 관리, 고객 보고서 작성 |
| Pain Point | 영어 메일 → 한국어 RCA 변환에 2~4시간 소요 |
| 기술 수준 | 인프라 전문가, 코딩은 제한적 |
| 사용 환경 | Windows 데스크탑, Outlook, Slack |

### Secondary Persona: 기술 리드

- RCA 보고서 검토 및 승인
- 팀 전체 장애 대응 현황 파악

---

## 3. 핵심 사용자 흐름 (User Journey)

```
[1] 웹앱 접속 → 티켓 목록 확인
        │
        ▼
[2] 티켓 선택 → 메일 스레드 확인
        │
        ▼
[3] "RCA 생성" 클릭 → AI가 분석 시작 (스트리밍)
        │
        ▼
[4] RCA 초안 검토 → 섹션별 수정/보완
        │
        ▼
[5a] "Word 내보내기" → .docx 다운로드
[5b] "Slack 공유" → 채널에 요약 전송
```

## 6. RCA 출력 품질 기준

AI가 생성하는 RCA는 다음 품질 기준을 만족해야 합니다:

| 섹션 | 품질 기준 |
|------|-----------|
| 요약 (Summary) | 3~5문장, 장애의 what/when/impact 포함 |
| 타임라인 (Timeline) | 시간순 정렬, 최소 5개 이벤트, 날짜/시간 포함 |
| 근본 원인 (Root Cause) | 기술적으로 정확, 메일에 언급된 근거 기반 |
| 조치 내역 (Resolution) | 실제 수행된 조치만 기술, 추측 배제 |
| 예방 대책 (Preventive Action) | 실행 가능한 구체적 권고사항 |
| 미해결 사항 (Open Questions) | 메일에서 답변되지 않은 질문 식별 |

### 전반 품질 원칙

- **한국어 자연스러움:** 번역투가 아닌 자연스러운 한국어
- **사실 기반:** 메일에 없는 내용을 만들어내지 않음 (Hallucination 방지)
- **출처 표시:** 각 섹션의 근거가 되는 메일 번호/날짜 참조 가능
- **일관된 톤:** 공식 보고서에 적합한 어조

## 7. Copilot SDK 활용 요구사항

앱 심사 기준 1번 (25% 가중치)에 대응:

| 활용 영역 | 구현 방식 | 깊이 |
|-----------|-----------|------|
| Agent 정의 | 역할별 에이전트 (파싱, 생성, 검토) | Multi-agent orchestration |
| Tool Calling | Graph API, 데이터 로드를 Tool로 등록 | 구조화된 도구 체인 |
| Prompt Engineering | 시스템 프롬프트 + Few-shot + 동적 컨텍스트 | 프로덕션 레벨 프롬프트 |
| Streaming | SSE로 생성 과정 실시간 전달 | 토큰 단위 스트리밍 |
| Context Management | 긴 메일 스레드 청킹 + 요약 전략 | 대용량 입력 처리 |
| Error Handling | 모델 오류/타임아웃 graceful 처리 | 사용자 피드백 포함 |

## 9. 제약사항 & 가정

### 제약사항

- 사용자는 코딩에 익숙하지 않으므로 모든 기능은 UI로 제공
- MVP는 1인 사용 환경 (멀티테넌트 불필요)
- 메일 데이터는 서버에 캐시하지 않음 (프라이버시)
- Azure 구독 비용은 월 $50 이내로 제한

### 가정

- 사용자는 Microsoft 365 라이선스 보유 (Outlook 접근)
- 벤더 서포트 메일은 영어로 작성됨
- 하나의 메일 스레드 = 하나의 서포트 티켓
- 메일 스레드 길이는 최대 50통
- Azure OpenAI GPT-4o 사용 가능 (할당량 확보)
