# 심사 기준 대응 전략 — Vendor Support RCA Copilot

본 문서는 앱 심사 7개 기준에 대해 구체적으로 어떻게 대응하는지 정리합니다.

---

## 1. Effective Use of Copilot SDK (25%)

> Copilot SDK가 앱의 가치에 핵심적인 역할을 합니까?

### 대응 전략

| 평가 항목 | 구현 방식 | 점수 확보 포인트 |
|-----------|-----------|-----------------|
| **프롬프트 설계** | 역할 기반 시스템 프롬프트, Few-shot 예시, 동적 컨텍스트 주입 | 프로덕션 레벨 프롬프트 엔지니어링 |
| **에이전트 설계** | Multi-Agent: MailParser → RCAGenerator → ReviewAssistant | 단일 에이전트가 아닌 체인 오케스트레이션 |
| **도구 호출** | fetchMailThread, parseMetadata, generateTimeline 등 도구 등록 | 구조화된 Tool 정의 + 결과 처리 |
| **컨텍스트 처리** | 긴 메일 스레드 청킹, 요약-상세 2단계 분석 | 토큰 한계 내 효과적 처리 |
| **스트리밍** | SSE 기반 토큰 단위 스트리밍, 섹션별 점진적 렌더링 | 사용자 체감 성능 향상 |

### 핵심 차별화

- SDK 없이는 앱의 핵심 기능(RCA 생성)이 불가능
- 단순 API 호출이 아닌, Agent 오케스트레이션으로 구현
- Tool calling을 통해 외부 데이터를 에이전트가 직접 가져오는 패턴

## 2. Productivity Impact & Problem Fit (18%)

> 명확한 대상 사용자에게 실질적 이점을 제공하면서 잘 정의된 생산성 문제를 해결합니까?

### 대응 전략

| 항목 | 증명 방식 |
|------|-----------|
| **대상 사용자** | MSP 엔지니어 (구체적 persona 정의) |
| **문제 크기** | RCA 1건당 2~4시간 → 15분 (75% 절감) |
| **문제의 빈도** | 월 10~15건 반복 발생 |
| **해결의 구체성** | 6개 섹션 구조화, 영→한 자동 변환 |
| **실증** | Before/After 시간 비교 데모 |

### 데모 시나리오

1. "이 50통 메일 스레드를 수동으로 읽고 RCA를 쓰면 2시간 걸립니다"
2. "우리 앱을 쓰면 버튼 한 번으로 15분 만에 완성됩니다"
3. 실제 샘플로 라이브 데모

## 3. Azure AI & Cloud Integration (18%)

> AI/모델 계층이 Microsoft Foundry / Azure OpenAI 위에서 동작합니까?

### 대응 전략

| 항목 | 구현 | 가점 요소 |
|------|------|-----------|
| **AI 모델** | Azure OpenAI GPT-4o (Microsoft Foundry 배포) | ✅ 필수 충족 |
| **서버리스 Backend** | Azure Functions (이벤트 기반) | 클라우드 네이티브 |
| **Frontend Hosting** | Azure App Service (Always On) | PaaS 활용 |
| **비밀 관리** | Azure Key Vault + Managed Identity | Best Practice |
| **모니터링** | Application Insights | 네이티브 통합 |
| **데이터** | Azure Table Storage | Azure 서비스 활용 |
| **인증** | Microsoft Entra ID | Azure 생태계 통합 |

### 감점 회피

- ❌ Azure 서비스 단순 끼워넣기 아님 → 모든 서비스가 유기적으로 연결됨
- ❌ 다른 클라우드 AI 사용하지 않음 → Azure OpenAI 전용
- ✅ Managed Identity로 서비스 간 인증 (API Key 미노출)

## 4. Functionality & Technical Execution (16%)

> 엔드투엔드로 작동합니까? 코드 품질, 아키텍처, 오류 처리는?

### 대응 전략

| 항목 | 구현 |
|------|------|
| **E2E 동작** | 티켓 선택 → RCA 생성 → 검토 → 내보내기 전 과정 동작 |
| **코드 품질** | TypeScript strict mode, ESLint, 일관된 프로젝트 구조 |
| **아키텍처** | 프론트/백 분리, 서버리스, 에이전트 레이어 분리 |
| **오류 처리** | API 에러 → 사용자 친화적 메시지, 재시도 로직, 타임아웃 |
| **성능** | 스트리밍 응답, 로딩 상태 표시, 비동기 처리 |

### 코드 구조 원칙

```
frontend/        → UI 레이어 (React + Fluent UI)
backend/         → API 레이어 (Azure Functions)
  └── agents/    → Copilot SDK Agent 정의
  └── tools/     → Agent가 사용하는 도구
  └── services/  → 비즈니스 로직
shared/          → 공통 타입, 인터페이스
```

## 5. User Experience & Workflow Design (12%)

> 사용자 저항이 낮은 UI, 자연스러운 AI 통합을 갖추었습니까?

### 대응 전략

| 항목 | 구현 |
|------|------|
| **저항이 낮은 UI** | Fluent UI (Microsoft 디자인 시스템), 익숙한 패턴 |
| **자연스러운 AI 통합** | 스트리밍으로 AI가 "타이핑하는" 느낌, 로딩 아닌 진행 표시 |
| **지연 처리** | 스켈레톤 UI + 진행 표시바 + 섹션별 점진 렌더 |
| **오류 처리** | 토스트 알림 + 재시도 버튼 + 에러 상세 (접기) |
| **사용자 주도권** | AI 결과를 항상 수정 가능, "재생성" 옵션 제공 |
| **접근성** | Fluent UI ARIA 기본 지원, 키보드 네비게이션 |

### UX 핵심 원칙

1. **AI는 도우미, 사용자가 최종 결정자** — 항상 수정 가능
2. **진행 상황 투명** — 무엇을 하고 있는지 실시간 표시
3. **3클릭 이내** — 메인 플로우 완료까지 최소 상호작용
4. **실패 시 복구 용이** — 이전 상태로 돌아갈 수 있음

## 6. Responsible AI, Security & Trust (6%)

> AI 투명성, Human-in-the-loop, 데이터 프라이버시를 갖추었습니까?

### 대응 전략

| 항목 | 구현 |
|------|------|
| **AI 투명성** | "AI가 생성한 초안입니다" 라벨 표시, 모델명/버전 표기 |
| **Human-in-the-loop** | RCA 자동 전송 없음, 반드시 사용자 검토 후 내보내기/공유 |
| **Hallucination 완화** | "메일에 근거한 내용만 작성" 시스템 프롬프트, 출처 참조 표시 |
| **데이터 프라이버시** | 메일 내용 서버 캐시 안 함, 세션 종료 시 삭제 |
| **프롬프트 인젝션** | 사용자 입력 sanitize, 시스템 프롬프트와 사용자 컨텍스트 분리 |
| **시크릿 관리** | .env (dev) / Key Vault (prod), 코드에 하드코딩 없음 |

### 보안 체크리스트

- [ ] 모든 API Key는 환경변수로만 접근
- [ ] .env 파일은 .gitignore에 포함
- [ ] HTTPS 전용 통신
- [ ] Bearer Token 인증
- [ ] CORS origin 제한
- [ ] 입력 길이 제한 (메일 최대 50통, 본문 10,000자)

## 7. Innovation & Originality (5%)

> 기존 서비스의 단순 복제가 아닌, AI를 새롭고 창의적으로 적용했습니까?

### 차별화 포인트

| 기존 도구 | 우리 앱의 차별점 |
|-----------|----------------|
| ChatGPT에 메일 붙여넣기 | 구조화된 Agent 체인, 표준 RCA 템플릿 자동 적용 |
| 일반 번역 도구 | 기술 컨텍스트 이해 + 보고서 형식 변환 동시 수행 |
| 기존 RCA 도구 | 메일 기반 자동 추출 (수동 입력 불필요) |
| Copilot 단독 사용 | 도메인 특화 프롬프트 + 도구 체인 + 워크플로우 통합 |

### 혁신 요소

1. **도메인 특화 Agent:** MSP 벤더 서포트라는 구체적 도메인에 최적화
2. **영→한 기술 보고서 변환:** 단순 번역이 아닌 보고서 형식 자동 구조화
3. **메일 → 보고서 파이프라인:** 비정형 이메일에서 정형 문서를 자동 생성하는 새로운 패턴
4. **Human-AI 협업 워크플로우:** AI 초안 + 사람 검토의 최적 밸런스

6. **(60초)** 기술 설명: Copilot SDK Agent 구조, Azure 아키텍처
